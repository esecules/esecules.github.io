UBC Course Scheduler

Specialties Used

Python

Docker

  • I worked as part of a team to develop a more user-friendly way for UBC students to plan their course schedules.
  • The goal was to build a tool where a student can enter the courses he/she needs and then be able to play around with which sections of the course fit their schedule.
  • For the next release we plan to add an auto-scheduler which uses a genetic algorithm to find a good schedule based on input criteria like walking distance and preferred hours.
  • I am responsible for the backend service that obtains the course information to be presented on the front end. When a user enters a course to add to their timetable, the client makes an ajax call back to my server's REST API. From there the server gathers the course information, packages it nicely in JSON and relays it back to the client.
  • I chose to use Docker to package up the backend because I would be able to have a consistent and controlled environment for development and it would be easy to deploy to AWS once we get to that stage

Pong ASIC

Specialties Used

  • My lab partner and I designed a four player version of pong in VHDL that can be played on an Altera DE2 board with a VGA display
  • I was responsible for designing the state machine which was responsible for moving the players and the puck, detecting and reacting to collisions, and keeping score
  • One of the main challenges of this project was hit detection. The problem was that whenever the puck hit one of the paddles, it stick to that paddle until the user changed its direction
    • This turned out to be a timing issue between the hit detection module and the state machine. To solve it I converted the hit detection module from a sequential circuit to a combinatorial one.
